The filter language this blog use

You may already seen the search box on the left of this blog. In fact I think you got here because you typed in title:filter && tags:blog.

The language support logic operators and brackets. In order to search for this page, as well as the main page, you may use this:

( title:filter && tags:blog ) || url:'index.html'

All spaces, except in 'strings', is optional. The above code could also been written as:

(title:filter&&tags:blog)||url:'index.html'

The interpreter excute the right operator first, so the code above can also been write as this to be shorter:

url:'index.html'||title:filter&&tags:blog

There are 4 basic logic operator in this language. && is the AND operator, || is the OR operator, ^ is the XOR operator and ! is the NOT operator. For &&, || and ^, you put two expression, in the left and in the right and it will do things you expect. For !, you put a expression in the right.

There are 6 test operators, they are :, =, <, >, <= and >=. You use these operators in forms of attr:string, where attr is the name of an attribute of one post, and string is the string you want to test.

For the : operator, it check whether the test string is in the attr.

This language support comparation of dates, for example date >= 2016-01-01 will give you all posts that is written after 2015. It can be shorten as date>=2016-.

If your string include only /[a-zA-Z0-9,\-]/, you can omit the 's. Otherwise you need to use 's to cover it up.

For attr, you can use title ( name means the same thing ), in ( tags means the same thing ) and date. ( By far )
